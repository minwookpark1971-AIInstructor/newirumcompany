### 1. apply.html 페이지의 희망일정을 admin 페이지에서 등록할 수 있도록 구현

#### Admin 페이지 구현 상태
- ✅ **탭 추가**: "일정 관리" 탭이 추가됨 (`data-tab="schedules"`)
- ✅ **섹션 추가**: `#schedules-section` 섹션이 구현됨
- ✅ **일정 추가 버튼**: `#add-schedule-btn` 버튼이 있음
- ✅ **일정 목록 테이블**: `#schedules-table` 테이블이 구현됨
- ✅ **모달**: `#schedule-modal` 모달이 구현됨
- ✅ **localStorage 저장**: `courseSchedules` 키로 저장됨

#### 함수 구현 상태
- ✅ `loadSchedules()`: localStorage에서 일정 로드
- ✅ `renderSchedules()`: 일정 목록 렌더링
- ✅ `addSchedule()`: 일정 추가 모달 열기
- ✅ `editSchedule(id)`: 일정 수정
- ✅ `deleteSchedule(id)`: 일정 삭제
- ✅ `saveSchedule()`: 일정 저장

#### Apply 페이지 구현 상태
- ✅ **입력 필드 변경**: `#preferredSchedule`이 `<select>`로 변경됨
- ✅ **일정 로드 함수**: `loadSchedules()` 함수가 구현됨
- ✅ **일정 필터링**: 활성화된 일정만 표시 (`s.active` 필터)
- ✅ **날짜 포맷팅**: 한국어 형식으로 표시 (`toLocaleDateString('ko-KR')`)

#### 데이터 흐름 검증
```
Admin 페이지 → localStorage.setItem('courseSchedules') 
→ Apply 페이지 → localStorage.getItem('courseSchedules') 
→ Select 옵션으로 표시
```

### 2. 신청자 정보 페이지에 과정 가격 표시 (admin에서 과정별 금액 등록)

#### Admin 페이지 구현 상태
- ✅ **탭 추가**: "과정 관리" 탭이 추가됨 (`data-tab="courses"`)
- ✅ **섹션 추가**: `#courses-section` 섹션이 구현됨
- ✅ **과정 목록 테이블**: `#courses-table` 테이블이 구현됨

- ✅ **과정 선택 시 호출**: 과정 선택 시 `updateCoursePrice()` 호출됨
- ✅ **Step 2 이동 시 호출**: Step 2로 이동 시 `updateCoursePrice()` 호출됨

#### 데이터 흐름 검증

Admin 페이지 → localStorage.setItem('coursePrices', {slug: price})

  - ✅ 상담 일정 목록 테이블: `#consult-schedules-table`
  - ✅ 모달: `#consult-schedule-modal`
  - ✅ 시간 선택 필드: `#consult-schedule-time` (Select)
- ✅ **localStorage 저장**: `consultSchedules` 키로 저장됨

#### 함수 구현 상태
- ✅ `renderConsultSchedules()`: 상담 일정 렌더링
- ✅ `addConsultSchedule()`: 상담 일정 추가 모달 열기
- ✅ `editConsultSchedule(id)`: 상담 일정 수정
- ✅ `deleteConsultSchedule(id)`: 상담 일정 삭제
- ✅ `saveConsultSchedule()`: 상담 일정 저장 (날짜 + 시간)

#### Apply 페이지 구현 상태
- ✅ **URL 파라미터 감지**: `type=instructor&action=consult` 파라미터 감지
- ✅ **모드 설정**: `isConsultMode = true` 설정
- ✅ **라벨 변경**: `updateScheduleLabelForConsult()` 함수로 라벨 변경
- ✅ **일정 로드**: `consultSchedules`에서 활성화된 일정만 로드
- ✅ **일정 표시**: 날짜와 시간을 함께 표시 (`${dateStr} ${schedule.time}`)
- ✅ **값 형식**: `schedule.date|schedule.time` 형식으로 저장

#### 데이터 흐름 검증
```
Admin 페이지 → localStorage.setItem('consultSchedules', [{date, time, active}])
→ Apply 페이지 (?type=instructor&action=consult) 
→ localStorage.getItem('consultSchedules')
→ 활성화된 일정만 필터링
→ Select 옵션으로 표시 (날짜 + 시간)
→ 선택 시 "날짜|시간" 형식으로 저장
```

## ⚠️ 발견된 잠재적 이슈

### 1. 날짜 형식 일관성
- **이슈**: Admin에서 저장할 때 `YYYY-MM-DD` 형식으로 저장되지만, Apply 페이지에서 표시할 때 `toLocaleDateString('ko-KR')`로 변환됨
- **영향**: 낮음 - 정상 작동하지만, 날짜 파싱 시 주의 필요
- **상태**: ✅ 정상 작동 확인

### 2. 상담 일정 시간 선택
- **이슈**: Admin 모달에서 시간 선택이 Select로 되어 있음 (09:00 ~ 20:00)
- **영향**: 없음 - 정상 작동
- **상태**: ✅ 정상 작동 확인

### 3. 일정 활성화 상태
- **이슈**: 비활성화된 일정은 Apply 페이지에서 표시되지 않음
- **영향**: 없음 - 의도된 동작
- **상태**: ✅ 정상 작동 확인
